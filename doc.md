Vim学習CUIアプリ「VimMaster (仮)」要件定義書
1. プロジェクト概要
Vimの基本操作から高度な編集コマンドまでを、実際に手を動かしながら習得するためのターミナル専用学習プラットフォーム。

コンセプト: 「vimtutor よりも楽しく、ゲーム感覚で指に覚えさせる」

開発言語/フレームワーク: Go / Bubble Tea (Charm ecosystem)

2. ターゲットユーザー
Vimを使い始めたが、hjkl移動だけで挫折しそうな初心者。

マウス操作を卒業し、編集効率を劇的に上げたい開発者。

d2w や ci" といったコンボコマンドを使いこなしたい中級者。

3. 機能要件
3.1 Vimエミュレーション機能
モード遷移: Normalモードを基本とし、Insertモード、Visualモードへの切り替えをサポートする。

キーキャプチャ: bubbletea の tea.KeyMsg を利用し、Vim特有のキーバインド（dw, y$, Gなど）をシミュレートする。

仮想バッファ: 画面上に表示されるテキストを内部で管理し、コマンドに応じて動的に書き換える。

3.2 ステージ・課題システム
ステージ構成:

基礎: h, j, k, l による移動。

編集: x, i, a, o 等によるテキスト追加・削除。

単語操作: w, e, b と d, c の組み合わせ。

ジャンプ: /, f, t, %, gg, G 等。

クリア判定: 内部バッファの状態が「期待されるテキスト状態」と一致した瞬間にクリアとする。

手数制限（任意）: 最小ストローク数（Vimの真髄）でクリアすると高評価を与える。

3.3 ユーザーインターフェース (TUI)
ヘッダー: 現在のステージ名、クリア条件、進捗率の表示。

メインエリア: 練習用のテキストバッファを表示（シンタックスハイライトがあれば尚良し）。

ステータスライン: 現在のモード（NORMAL/INSERT）、入力中のキー（d を押した後の待機状態など）を表示。

ヘルプ/ヒントエリア: 詰まった時に表示される操作ガイド。

4. 非機能要件
パフォーマンス: 入力遅延（レイテンシ）を最小限に抑え、本物のVimに近い操作感を実現する。

拡張性: 新しい課題（JSONやYAML形式）を簡単に追加できる設計にする。

ポータビリティ: go build で単一バイナリとして配布可能にする。

5. 技術スタック詳細
TUI Framework: Bubble Tea (全体のライフサイクル管理)

Styling: Lip Gloss (文字色、枠線、レイアウト)

Components: Bubbles (テキスト入力やビューポートの再利用)

7. 開発ロードマップ (MVP順)
Phase 1: Bubble Tea で「文字が表示され、hjkl でカーソルが動く」だけのモックを作成。

Phase 2: NormalモードとInsertモードの切り替えを実装。

Phase 3: dw や x などの基本的な編集コマンドの実装。

Phase 4: 正解判定ロジックと、複数ステージの遷移機能を実装。
